# AstraGlyph

**Версия:** 0.2.0  
**Язык:** C++20  
**Лицензия:** (не указана)

---

## 1. Общее описание

**AstraGlyph** — экспериментальный ASCII-рендерер трёхмерных сцен с трассировкой лучей. Проект визуализирует 3D-геометрию символами из набора ASCII, где яркость (luminance) каждой ячейки отображается определённым символом — от тёмных точек до ярких «@» или «#». Рендерер поддерживает загрузку внешних моделей в формате **Wavefront OBJ** с материалами **MTL** и текстурами, программную генерацию примитивов (плоскость, куб, сфера), многоуровневое освещение (направленное, точечное, площадное), тени, мягкие тени, отражения, адаптивное и временное сэмплирование, **blue-noise джиттер**, **shape-aware глифы** с edge enhancement, **specular/rim light** в отдельном темпоральном канале, **checkerboard interleaved sampling**, а также ускорение трассировки через **BVH** (Bounding Volume Hierarchy) с **SAH** и **SoA**-форматом данных.

Главное окно создаётся через **SDL3** и отрисовывает каждый ASCII-символ в виде набора цветных прямоугольников (глифов), формируя итоговое изображение. Весь конвейер рендеринга работает с плавающей точкой (`Vec3`-радианс) и не зависит от графического API — SDL используется только для вывода на экран и ввода.

## 2. Архитектура

Проект построен по модульному принципу с чётким разделением ответственности:

- **ASCII-символ** — минимальная единица изображения. Внутри ячейки не создаётся пиксельный framebuffer.
- Вся геометрия проходит через единый конвейер `Mesh` → `Triangle`.
- `RayTracer` не зависит от SDL, окон, UI и ASCII-вывода.
- `Renderer` не зависит от деталей SDL.
- `Scene` не зависит от UI.

### 2.2. Ключевые компоненты

| Компонент | Описание |
|-----------|----------|
| `Application` | Главный цикл `update → render`. Управляет окном, камерой, сценой, рендерером и UI-панелью. |
| `Window` | Обертка над SDL3: создание окна, рендерера, очистка кадра, рисование прямоугольников, present. Поддерживает прямую запись в pixel buffer. |
| `Input` | Абстракция ввода: клавиатура (WASD, цифры, F1/F3 и др.), мышь (захват/осмотр, колесо, клики), состояния «нажато/отпущено». |
| `Camera` | Перспективная камера с полётом от первого лица. Поддерживает yaw/pitch, FOV, aspect ratio, экспоненциальное ускорение и трение. Генерирует первичные лучи. |
| `Scene` | Контейнер для `Mesh`, `Material`, `Light`, `Triangle`. Предоставляет плоский список треугольников для трейсера. |
| `SceneLoader` | Загружает сцены из **JSON** (камера, настройки рендера, материалы, свет, объекты). При ошибках или отсутствии файла выполняет fallback на встроенную сцену по умолчанию. |
| `Mesh` | Набор треугольников с материалом. Поддерживает трансформации: `translate`, `scale`, `rotate`. |
| `Triangle` | Геометрический треугольник с вершинными позициями, нормалями, UV. Реализует пересечение луча по Моллеру–Трумбору (Möller–Trumbore). Кэширует рёбра для ускорения. |
| `MeshFactory` | Генерация примитивов: `plane`, `cube` (с заданными размерами), `uvSphere`. |
| `ObjLoader` | Загрузчик **Wavefront OBJ** (`v`, `vt`, `vn`, `f`, `o`, `g`, `usemtl`, `mtllib`). Поддерживает многоугольные грани (триангуляция веером), относительные индексы, автоматическую генерацию нормалей. Загружает материалы **MTL** и связанные текстуры (через `stb_image`). |
| `RayTracer` | Высокоуровневый интерфейс трассировки. Использует BVH при включённом флаге или brute-force перебор. |
| `Bvh` | Иерархия ограничивающих объёмов с построением по **SAH** (Surface Area Heuristic). Использует 8-бинное разбиение центроидов, `std::stable_partition`, **SoA**-хранение треугольников для cache-friendly доступа. Поддерживает **packet tracing** для пакетов лучей. |
| `Integrator` | Интегратор освещённости. Вычисляет `traceRadiance` с учётом прямого освещения, теней, мягких теней (area light sampling), отражений (recursive reflections до `maxBounces`). Поддерживает сэмплирование текстур материалов. |
| `Renderer` | Главный рендер-конвейер. Разбивает кадр на тайлы, запускает многопоточную трассировку через `ThreadPool`, управляет временным накоплением (TAA с тремя каналами), адаптивным сэмплированием, exposure/gamma, **shape-aware глифами**, **checkerboard interleaved sampling**, specular boost. |
| `AsciiMapper` | Преобразование радианса в яркость (Rec.709) и выбор ASCII-символа. Применяет exposure и gamma-коррекцию. Поддерживает **shape-aware глифы** (направленные ramp по градиенту luminance через Sobel 3×3) и **edge enhancement** (unsharp mask). |
| `Sampler` | Стратифицированное сэмплирование (2/4/8/16 сэмплов с фиксированными паттернами + **blue-noise джиттер** через перестановочную таблицу). Генерация сидов на основе хэш-функций. |
| `TemporalAccumulator` | Временное накопление (TAA) с **тремя независимыми каналами**: `direct` (α=0.6), `indirect` (α=0.15), `specular` (α=0.8). Каждый канал использует собственную скорость сходимости EMA. Поддерживает checkerboard-совместимое чтение. |
| `AsciiFramebuffer` | Сетка ячеек `AsciiCell` (glyph, fg, bg, radiance, luminance, depth, normal, accumulation state). |
| `RenderSettings` | Структура всех настроек рендера: размер сетки, сэмплы, тени/отражения/BVH, exposure, gamma, цвет/монохром, ramp-режим, backface culling, debug view, профилирование, shape-aware glyphs, rim light, specular boost, checkerboard sampling и т.д. |
| `UIPanel` | Интерактивная боковая панель с чекбоксами, слайдерами, статистикой (FPS, grid, samples, треугольники, позиция камеры) и справкой по горячим клавишам. Управляется мышью (клики, колесо, скроллбар). |
| `ThreadPool` | Постоянный пул потоков для многопоточного рендеринга. Переиспользуется между кадрами, использует atomic-счётчики и task-based модель. |

### 2.3. Оптимизации производительности

#### Кэширование рёбер треугольников
В `Triangle` вычисляемые в `intersect()` рёбра `edge1 = v1 - v0` и `edge2 = v2 - v0` кэшируются при создании. Метод `computeCachedEdges()` вызывается в `MeshFactory`, `ObjLoader` и при трансформациях `Mesh`. Это исключает повторные вычисления для каждого луча.

#### SAH BVH + SoA
- Построение по **Surface Area Heuristic** с 8-бинным анализом центроидов и префиксными проходами для O(n) оценки split-точек.
- Треугольники в листьях хранятся в **Structure of Arrays (SoA)** формате (`TriangleSoA`) для улучшения локальности кэша при SIMD-обработке.
- **Packet tracing**: `Bvh::intersectPacket()` для одновременной трассировки нескольких лучей.

#### SIMD-оптимизация пересечений
При включённой опции `ASTRAGLYPH_ENABLE_SIMD` используется пакетная обработка пересечений лучей с треугольниками:
- **AVX2 + FMA**: до 8 треугольников параллельно
- **SSE3**: до 4 треугольников параллельно
- **Fallback**: scalar версия (гарантированная совместимость)

Подробнее см. `docs/SIMDOptimization.md`.

#### Многопоточность
Рендерер разбивает кадр на горизонтальные тайлы и обрабатывает их в пуле потоков (`ThreadPool`). Локальные метрики агрегируются после `waitAll()` без mutex в hot path.

Подробнее см. `docs/CPUMultithreading.md`.

#### Оптимизации сборки
Release-сборка использует флаги `-O3` для GCC/Clang и `/O2` для MSVC.

## 4. Сборка и запуск

### 4.1. Требования

- CMake 3.25+
- Компилятор C++20 (GCC, Clang, MSVC)
- (Опционально) Ninja или Make

Проект содержит пресеты для `MinGW Makefiles` и `Ninja`. Доступны конфигурации **Debug** и **Release**.

```powershell
cmake --preset debug

ctest --preset debug --output-on-failure
```

#### Release (Makefiles)

```powershell
cmake --preset release
cmake --build --preset release
```

#### Debug (Ninja)

```powershell
cmake --preset debug-ninja
cmake --build --preset debug-ninja
ctest --preset debug-ninja --output-on-failure
```

#### Release (Ninja)

```powershell
cmake --preset release-ninja
cmake --build --preset release-ninja
```

### 4.3. PowerShell-скрипты (Windows)

В каталоге `scripts/` находятся удобные обёртки:

```powershell
# Конфигурация
.\scripts\configure.ps1 -Preset debug

# Сборка
.\scripts\build.ps1 -Preset debug

# Полная очистка
.\scripts\clean_build.ps1 -Preset debug   # или -Preset all

Эквивалентные `.sh` скрипты присутствуют для Unix-систем.

cmake .. -G "Ninja" -DCMAKE_BUILD_TYPE=Release
ninja
```

Исполняемый файл: `build/debug/astraglyph.exe` (или `./astraglyph` на Unix).

### 5.3. Управление панелью (мышь)

- **ЛКМ** по чекбоксу/кнопке — переключить.
- **ЛКМ** / **ПКМ** по слайдеру — увеличить/уменьшить значение.
- **Колесо мыши** — прокрутка панели (при наличии скроллбара).
- Курсор над правой частью экрана (где панель) блокирует захват мыши камерой.

#### Объекты (`objects`)

| Тип | Параметры | Описание |
|-----|-----------|----------|
| `plane` | `size: [width, y, depth]` | Плоскость на XZ |
| `cube` | `size: [x, y, z]` или `size: scalar` | Куб с заданными размерами |
| `sphere` | `radius`, `segments`, `rings` | UV-сфера |
| `obj` | `path` — путь к `.obj`; `mtlPath` — опционально, путь к `.mtl` | Внешняя модель Wavefront OBJ |

Все объекты поддерживают `transform` с полями `position`, `scale`, `rotation` (в радианах, yaw/pitch/roll).

Материалы объектов задаются через строковое имя, указанное в секции `materials`. Материал может быть задан явно через поля `albedo`, `roughness` и т.д., или загружен из MTL-файла через `mtlPath`.

### 6.2. Загрузка моделей OBJ и материалов MTL

Рендерер поддерживает стандарт **Wavefront OBJ**:

- Вершины (`v x y z`)
- Текстурные координаты (`vt u v`)
- Нормали (`vn x y z`)
- Грани (`f v/vt/vn v/vt/vn ...`) — многоугольники автоматически триангулируются веером
- Объекты (`o`), группы (`g`), материалы (`usemtl`, `mtllib`)
- Относительные индексы (отрицательные числа)

Если нормали отсутствуют, они вычисляются автоматически по грани. Если загрузка OBJ не удалась, вместо модели подставляется маленький куб (`0.1`).

**Поддержка MTL и текстур:**
- Загрузка материалов из `.mtl` файлов (`newmtl`, `Kd`, `Ke`, `Ns`, `d`, `Tr`)
- Текстуры: `map_Kd` / `map_Ka` (albedo), `map_Metalness` / `map_metallic` (metallic), `map_roughness` / `map_Pr` / `map_Ns` (roughness)
- Загрузка через `stb_image` (PNG, JPG и др.)
- Текстуры сэмплируются в линейном пространстве для корректного освещения

**Пример использования в коде:**

```cpp
Mesh mesh = ObjLoader::loadFromFile("assets/models/test_cube.obj", ObjLoadOptions{0});
scene.addMesh(std::move(mesh));
```

### 6.3. Рендер-конвейер

1. **Генерация лучей** — для каждой ячейки сетки `gridWidth × gridHeight` камера выпускает первичные лучи.
2. **Сэмплирование** — внутри ячейки используется стратифицированное сэмплирование с **blue-noise джиттером** (2/4/8/16 сэмплов). При включённом `adaptiveSampling` ячейки с высокой дисперсией яркости получают дополнительные сэмплы до `maxSamplesPerCell`.
3. **Checkerboard sampling** — при `checkerboardSampling` + `temporalAccumulation` в статике каждый кадр трассируется только половина клеток (шахматный порядок), вторая половина сохраняется из предыдущего кадра. При движении камеры автоматически отключается.
4. **Трассировка** — `RayTracer` ищет ближайшее пересечение через BVH (с SAH, SoA, packet tracing) или brute-force.
5. **Интеграция** — `Integrator` вычисляет освещённость:
   - Фоновое освещение (`ambientStrength`)
   - **Прямое освещение** (diffuse) от всех источников света
   - **Specular** — Blinn-Phong блики с отдельным накоплением
   - **Rim light** — Fresnel-подсветка рёбер объектов (если `enableRimLight`)
   - **Текстуры** — сэмплирование albedo/metallic/roughness по UV
   - **Тени** — трассировка лучей от точки попадания к источнику света
   - **Мягкие тени** — для `Area` источников выполняется множественная выборка точек на площадке (`shadowSamples`)
   - **Отражения** — рекурсивная трассировка отражённых лучей до `maxBounces` с учётом `reflectivity` материала
6. **Tone mapping** — filmic exposure (`1 - exp(-radiance * exposure)`) + gamma-коррекция.
7. **Shape-aware ASCII-маппинг** — `AsciiMapper` вычисляет градиент luminance (Sobel 3×3) и выбирает направленный символ (horizontal `#=-:. `, vertical `#|!:. `, diagonal `#/\:. `) на границах объектов. Edge enhancement (unsharp mask) усиливает контраст на высокоградиентных пикселях.
8. **Временное накопление (TAA)** — при `temporalAccumulation` результаты накапливаются раздельно по трём каналам: `direct` (α=0.6), `indirect` (α=0.15), `specular` (α=0.8). Specular-канал усиливается (`specularBoost`) после накопления, сохраняя чёткость бликов.
9. **Вывод** — `Application` рисует каждую ячейку цветными прямоугольниками через SDL.

### 6.4. Режимы отображения

#### Glyph Ramp

| Режим | Набор символов |

| `Classic` | `@#$S%?&*iI:-. ` |
| `Filled` | Полная заливка ячейки цветом (без символа) |

Выбирается через `renderSettings.glyphRampMode` в JSON-сцене или клавишей `6`.

#### Debug View

| Режим | Описание |
|-------|----------|
| `Off` | Обычный рендеринг |
| `Albedo` | Отображение базового цвета (с текстурами) |

### 6.5. Многопоточность

Рендерер разбивает кадр на горизонтальные тайлы и обрабатывает их в отдельных потоках через постоянный `ThreadPool`. Количество потоков:

- `threadCount == 0` — автоопределение (`hardware_concurrency`)
- `threadCount > 0` — фиксированное число (до 64)

Многопоточность включается флагом `enableMultithreading`. На сетках ≤ 8 строк используется single-threaded режим.

### 6.6. SIMD-оптимизация

Включается опцией CMake `ASTRAGLYPH_ENABLE_SIMD=ON`:
- Автоматический выбор между AVX2+FMA и SSE3 в зависимости от компилятора
- Идентичность результатов со scalar версией (тот же алгоритм Möller–Trumbore, те же пороги)
- Используется в BVH leaf nodes при ≥4 треугольников в узле
- Требуется выровненная по 32 байта память для batch данных

### 6.8. Фазы развития рендерера (v0.2.0)

| Фаза | Название | Описание |
|------|----------|----------|
| **1** | **TAA + Blue-noise sampling** | Разделение temporal accumulation на два канала (`direct` α=0.6, `indirect` α=0.15). Blue-noise джиттер вместо белого шума для уменьшения паттерн-артефактов. |
| **2** | **Direct/indirect EMA split** | Независимые скорости сходимости для прямого и отражённого света. Direct стабилизируется быстро, indirect сглаживается медленно. |
| **3** | **Contrast-aware glyph mapping + edge enhancement** | Sobel-градиент на luminance для выбора направленных глифов (`#=-`, `#\|`, `#/\`). Unsharp-mask усиление рёбер. Поле `normal` в `AsciiCell`. |
| **4** | **Specular exaggeration + rim light** | Третий канал `specular` (α=0.8) в `TemporalAccumulator`. Blinn-Phong блики и Fresnel rim light отдельно от diffuse. `specularBoost` усиливает блики после накопления без размытия. |
| **5** | **Checkerboard / interleaved sampling** | В статике каждый кадр рендерит только половину клеток (шахматный порядок), чередуя чётность по `frameIndex`. Эквивалентно удвоению разрешения без увеличения числа лучей. Автоотключение при движении камеры. |

### 6.7. ASCII Glyph Rendering (динамические глифы)

Каждый ASCII-символ отрисовывается через пиксельную маску, сгенерированную на лету из встроенного шрифта **Terminus** (monospace, TTF) с помощью библиотеки **stb_truetype**:

- При изменении размера ячейки (`cellWidth` × `cellHeight`) `GlyphAtlas` перестраивает кэш масок для всех используемых символов.
- Маска — плоский массив `cellW × cellH` байт (`0` = фон, `1` = глиф).
- Растеризация: `stbtt_GetCodepointBitmap` с масштабом `cellH × 0.8`, центрирование в ячейке, порог бинаризации `128`.
- Шрифт Terminus распространяется под лицензией **SIL Open Font License 1.1**.

Это заменяет прежние жёстко заданные 4×5 битовые паттерны и позволяет адаптировать форму символов под любое разрешение сетки.

### 6.8. Метрики производительности

`RenderMetrics` (отображаются в `UIPanel` и заголовке окна):

- FPS, время кадра, размер сетки, среднее число сэмплов на ячейку
- Количество первичных, теневых, отражённых и вторичных лучей
- Число треугольников, BVH-нодов, листьев
- Позицию камеры
- Состояние всех ключевых фич (тени, отражения, BVH, цвет и т.д.)
- Количество накопленных кадров (temporal accumulation)
- Использованное число потоков
- **Профилирование** (при `F3`): время BVH traversal, пересечений треугольников, шейдинга, вывода и общее время рендера

## 8. Как начать работу

1. **Клонируйте репозиторий** и перейдите в корневую папку.
2. **Соберите проект** через CMake preset или PowerShell-скрипты.
3. **Запустите** `astraglyph.exe` (или `./astraglyph`).
4. **Управляйте камерой** с помощью `WASD` + `Shift`/`Ctrl`, осмотр — зажатие ЛКМ.
5. **Экспериментируйте с настройками** через горячие клавиши (`1`–`6`, `[`, `]`, `-`, `=`, `Space`, `F3`).
6. **Создавайте свои сцены**: отредактируйте `assets/scenes/default_scene.json` или создайте новый JSON и измените путь в `Application.cpp` (по умолчанию загружается `samurai_girl_scene.json`).
7. **Добавляйте модели OBJ**: поместите `.obj` и `.mtl` (с текстурами) в `assets/models/` и добавьте объект типа `"obj"` в JSON-сцену. Если `mtlPath` не указан, MTL ищется рядом с OBJ автоматически.

## 9. Примечания и ограничения

- **Минимальная единица** — ASCII-символ. Нет субпиксельного рендеринга внутри ячейки.
- **Максимальное разрешение сетки** — 320×180 (ограничение `RenderSettings::validate`).
- **Максимум сэмплов** — 16 на ячейку (адаптивно — до 16).
- **Максимум отражений** — 8 bounces.
- **Цветовой вывод** — включается флагом `colorOutput`; при выключении изображение монохромное.
- **Temporal accumulation** — три независимых канала (direct/indirect/specular) с разными α.
- **Checkerboard sampling** — удваивает эффективное разрешение в статике за счёт чередования клеток.
- **SDL3** скачивается и собирается автоматически при первой конфигурации CMake (требуется интернет).
- **SIMD** опционален и требует включения через `ASTRAGLYPH_ENABLE_SIMD`.
- На сетках ≤ 8 строк многопоточность отключается автоматически.
