# AstraGlyph / AsciiTracer

**Версия:** 0.1.0  
**Язык:** C++20  
**Лицензия:** (не указана)

---

## 1. Общее описание

**AstraGlyph** (также известен как **AsciiTracer**) — это экспериментальный ASCII-рендерер трёхмерных сцен с трассировкой лучей. Проект визуализирует 3D-геометрию не привычными пикселями, а символами из набора ASCII, где яркость (luminance) каждой ячейки сетки отображается определённым символом — от тёмных точек до ярких «@» или «#». Рендерер поддерживает загрузку внешних моделей в формате **Wavefront OBJ**, программную генерацию примитивов (плоскость, куб, сфера), многоуровневое освещение (направленное, точечное, площадное), тени, мягкие тени, отражения, адаптивное и временное сэмплирование, а также ускорение трассировки через **BVH** (Bounding Volume Hierarchy).

Главное окно создаётся через **SDL3** и отрисовывает каждый ASCII-символ в виде набора цветных прямоугольников (глифов), формируя итоговое изображение. Весь конвейер рендеринга работает с плавающей точкой (`Vec3`-радианс) и не зависит от графического API — SDL используется только для вывода на экран и ввода.

## 2. Архитектура

Проект построен по модульному принципу с чётким разделением ответственности. Основные архитектурные правила:

- **ASCII-символ** — минимальная единица изображения. Внутри ячейки не создаётся пиксельный framebuffer.
- Вся геометрия проходит через единый конвейер `Mesh` → `Triangle`.
- Примитивы и OBJ-ассеты конвертируются в общий pipeline.
- `RayTracer` не зависит от SDL, окон, UI и ASCII-вывода.
- `Renderer` не зависит от деталей SDL.
- `Scene` не зависит от UI.

### 2.1. Структура каталогов

```

│   ├── geometry/      # Геометрические примитивы, фабрики, загрузчик OBJ

│   ├── platform/      # Оконная система, ввод, время (SDL3)

```

### 2.2. Ключевые компоненты

| Компонент | Описание |
|-----------|----------|
| `Application` | Главный цикл `update → render`. Управляет окном, камерой, сценой, рендерером и UI-наложениями. |
| `Window` | Обертка над SDL3: создание окна, рендерера, очистка кадра, рисование прямоугольников, present. |
| `Input` | Абстракция ввода: клавиатура (WASD, цифры, F1/F2 и др.), мышь (захват/осмотр), состояния «нажато/отпущено». |
| `Camera` | Перспективная камера с полётом от первого лица. Поддерживает yaw/pitch, FOV, aspect ratio, экспоненциальное ускорение и трение. Генерирует первичные лучи. |
| `Scene` | Контейнер для `Mesh`, `Material`, `Light`, `Triangle`. Предоставляет плоский список треугольников для трейсера. |
| `SceneLoader` | Загружает сцены из **JSON** (камера, настройки рендера, материалы, свет, объекты). Поддерживает fallback на дефолтную сцену при ошибках. |
| `Mesh` | Набор треугольников с материалом. Поддерживает трансформации: `translate`, `scale`, `rotate`. |
| `Triangle` | Геометрический треугольник с вершинными позициями, нормалями, UV. Реализует пересечение луча по Моллеру–Трумбору (Möller–Trumbore). |
| `MeshFactory` | Генерация примитивов: `plane`, `cube` (с заданными размерами), `uvSphere`. |
| `ObjLoader` | Загрузчик **Wavefront OBJ** (`v`, `vt`, `vn`, `f`, `o`, `g`, `usemtl`, `mtllib`). Поддерживает многоугольные грани (триангуляция веером), относительные индексы, автоматическую генерацию нормалей при отсутствии. |
| `RayTracer` | Высокоуровневый интерфейс трассировки. Использует BVH при включённом флаге или brute-force перебор. |
| `Bvh` | Иерархия ограничивающих объёмов (BVH) с построением по центроидам (axis-aligned, midpoint split). Ускоряет поиск пересечений в разы. |
| `Integrator` | Интегратор освещённости. Вычисляет `traceRadiance` с учётом прямого освещения, теней, мягких теней (area light sampling), отражений (recursive reflections до `maxBounces`). |
| `Renderer` | Главный рендер-конвейер. Разбивает кадр на тайлы, запускает многопоточную трассировку, управляет временным накоплением (`temporal accumulation`), адаптивным сэмплированием, exposure/gamma, выбором глифа. |
| `AsciiMapper` | Преобразование радианса в яркость (Rec.709) и выбор ASCII-символа по ramp (Classic / Dense / Blocks). Применяет exposure и gamma-коррекцию. |
| `Sampler` | Стратифицированное сэмплирование (2/4/8/16 сэмплов с фиксированными паттернами + джиттер). Генерация сидов на основе хэш-функций. |
| `AsciiFramebuffer` | Сетка ячеек `AsciiCell` (glyph, fg, bg, radiance, luminance, depth, accumulation state). |
| `RenderSettings` | Структура всех настроек рендера: размер сетки, количество сэмплов, включение теней/отражений/BVH, exposure, gamma, цвет/монохром, ramp-режим и т.д. |
| `DebugOverlay` | Отображает FPS, время кадра, количество лучей, треугольников, BVH-нодов, позицию камеры, состояние фич. |
| `SettingsPanel` | Панель с текущими значениями настроек и списком горячих клавиш. |

## 3. Зависимости

Проект использует **CMake ≥ 3.25** и компилятор с поддержкой **C++20**.

| Зависимость | Версия | Назначение | Способ поставки |
|-------------|--------|------------|-----------------|
| **SDL3** | 3.2.16 | Окно, события, рендерер 2D | Автозагрузка и сборка из исходников при отсутствии в системе |
| **nlohmann/json** | 3.11.3 | Парсинг JSON-сцен | Автозагрузка single-header |

SDL3 конфигурируется с отключёнными подсистемами (audio, camera, joystick, haptic, GPU, OpenGL, Vulkan и др.), оставляя только `VIDEO` и `RENDER`.

- CMake 3.25+

```powershell
cmake --preset release

```

#### Debug (Ninja)

```powershell
cmake --preset debug-ninja

ctest --preset debug-ninja --output-on-failure
```

#### Release (Ninja)

```powershell
cmake --preset release-ninja
```

Камера использует экспоненциальное ускорение и трение для плавного старта/остановки. Скорость, чувствительность мыши, ускорение и трение заданы в коде `Camera`.

### 5.2. Горячие клавиши рендера

| Клавиша | Действие |
|---------|----------|
| `F1` | Показать/скрыть **Debug Overlay** |
| `F2` | Показать/скрыть **Settings Panel** |
| `1` | Вкл/выкл **тени** |
| `2` | Вкл/выкл **мягкие тени** (area light sampling) |
| `3` | Вкл/выкл **отражения** |
| `4` | Вкл/выкл **адаптивное сэмплирование** |
| `5` | Вкл/выкл **BVH** ускорение |
| `[` | Уменьшить `samplesPerCell` (-1) |
| `]` | Увеличить `samplesPerCell` (+1) |
| `-` | Уменьшить качество вторичных лучей (soft shadow samples или max bounces) |
| `=` | Увеличить качество вторичных лучей |

Все изменения применяются в реальном времени. При изменении настроек, влияющих на изображение, происходит сброс временного накопления (temporal accumulation).

#### Объекты (`objects`)

| Тип | Параметры | Описание |
|-----|-----------|----------|
| `plane` | `size: [width, y, depth]` | Плоскость на XZ |
| `cube` | `size: [x, y, z]` или `size: scalar` | Куб с заданными размерами |
| `sphere` | `radius`, `segments`, `rings` | UV-сфера |
| `obj` | `path` — путь к `.obj` файлу | Внешняя модель Wavefront OBJ |

Все объекты поддерживают `transform` с полями `position`, `scale`, `rotation` (в радианах, yaw/pitch/roll).

Материалы объектов задаются через строковое имя, указанное в секции `materials`.

### 6.2. Загрузка моделей OBJ

Рендерер поддерживает стандарт **Wavefront OBJ**:

- Вершины (`v x y z`)
- Текстурные координаты (`vt u v`)
- Нормали (`vn x y z`)
- Грани (`f v/vt/vn v/vt/vn ...`) — многоугольники автоматически триангулируются веером
- Объекты (`o`), группы (`g`), материалы (`usemtl`, `mtllib`)
- Относительные индексы (отрицательные числа)

Если нормали отсутствуют, они вычисляются автоматически по грани. Если загрузка OBJ не удалась, вместо модели подставляется маленький куб (`0.1`).

**Пример использования в коде:**

```cpp
Mesh mesh = ObjLoader::loadFromFile("assets/models/test_cube.obj", ObjLoadOptions{0});
scene.addMesh(std::move(mesh));
```

### 6.3. Рендер-конвейер

1. **Генерация лучей** — для каждой ячейки сетки `gridWidth × gridHeight` камера выпускает первичные лучи.
2. **Сэмплирование** — внутри ячейки используется стратифицированное сэмплирование с джиттером (2/4/8/16 сэмплов). При включённом `adaptiveSampling` ячейки с высокой дисперсией яркости получают дополнительные сэмплы до `maxSamplesPerCell`.
3. **Трассировка** — `RayTracer` ищет ближайшее пересечение через BVH или brute-force.
4. **Интеграция** — `Integrator` вычисляет освещённость:
   - Фоновое освещение (`ambientStrength`)
   - Прямое освещение от всех источников света
   - **Тени** — трассировка лучей от точки попадания к источнику света
   - **Мягкие тени** — для `Area` источников выполняется множественная выборка точек на площадке (`shadowSamples`)
   - **Отражения** — рекурсивная трассировка отражённых лучей до `maxBounces` с учётом `reflectivity` материала
5. **Tone mapping** — exposure + gamma-коррекция.
6. **ASCII-маппинг** — яркость преобразуется в символ из выбранной ramp (Classic / Dense / Blocks).
7. **Временное накопление** — при `temporalAccumulation` результаты усредняются между кадрами, пока камера/настройки не изменились.
8. **Вывод** — `Application` рисует каждую ячейку цветными прямоугольниками через SDL, отображая глиф и фон.

### 6.4. Режимы отображения символов (Glyph Ramp)

| Режим | Набор символов |
|-------|----------------|
| `Classic` | `@#$S%?&*iI:-. ` |
| `Dense` | `` `'.,:^~";!i<>-?][}{1)(|/tfjrxnuvczXYUJCLQ0OZmwqpdbkhao*#MW&8%B@$ `` |
| `Blocks` | ` .:=*#@` |

Выбирается через `renderSettings.glyphRampMode` в JSON-сцене.

### 6.5. Многопоточность

Рендерер разбивает кадр на горизонтальные тайлы и обрабатывает их в отдельных потоках. Количество потоков:

- `threadCount == 0` — автоопределение (`hardware_concurrency`)
- `threadCount > 0` — фиксированное число (до 64)

Многопоточность включается флагом `enableMultithreading`.

### 6.6. Метрики производительности

`DebugOverlay` отображает в реальном времени:

- FPS и время кадра (мс)
- Размер сетки и среднее число сэмплов на ячейку
- Количество первичных, теневых и отражённых лучей
- Число треугольников, BVH-нодов и листьев
- Позицию камеры
- Состояние всех ключевых фич (тени, отражения, BVH, цвет и т.д.)
- Количество накопленных кадров (temporal accumulation)

## 8. Как начать работу

1. **Клонируйте репозиторий** и перейдите в корневую папку.
2. **Соберите проект** через CMake preset или PowerShell-скрипты.
3. **Запустите** `astraglyph.exe` (или `./astraglyph`).
4. **Управляйте камерой** с помощью `WASD` + `Shift`/`Ctrl`, осмотр — зажатие ЛКМ.
5. **Экспериментируйте с настройками** через горячие клавиши (`1`–`5`, `[`, `]`, `-`, `=`).
6. **Создавайте свои сцены**: отредактируйте `assets/scenes/default_scene.json` или создайте новый JSON и укажите путь в `Application.cpp` (по умолчанию загружается `default_scene.json`).
7. **Добавляйте модели OBJ**: поместите `.obj` файл в `assets/models/` и добавьте объект типа `"obj"` в JSON-сцену.

## 9. Примечания и ограничения

- **Минимальная единица** — ASCII-символ. Нет субпиксельного рендеринга внутри ячейки.
- **Максимальное разрешение сетки** — 320×180 (ограничение `RenderSettings::validate`).
- **Максимум сэмплов** — 16 на ячейку (адаптивно — до 16).
- **Максимум отражений** — 8 bounces.
- **Цветовой вывод** — включается флагом `colorOutput`; при выключении изображение монохромное.
- **SDL3** скачивается и собирается автоматически при первой конфигурации CMake (требуется интернет).
